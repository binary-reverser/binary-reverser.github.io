Android平台编译ARM汇编的两种姿势

AndroidStudio3.x集成环境中编译arm汇编源文件（以arm64为例）

build.gradle中设置目标二进制文件运行架构

defaultConfig {
ndk {
abiFilters ‘arm64-v8a’
}
}

CMakeLists.txt中的配置

使能汇编编译

set(can_use_assembler TRUE)
enable_language(ASM)

add_definitions(-DANDROID -DOC_ARM_ASM)

以静态库的方式单独编译汇编源文件

add_library( assembly-lib STATIC
src/main/assembly/syscall.s )

将静态库链接到目标共享库文件

target_link_libraries( # Specifies the target library.
native-lib
assembly-lib
# Links the target library to the log library
# included in the NDK.
)

通过android-ndk工具链手动编译汇编源文件（以arm32为例）

Application.mk中设置目标二进制文件运行架构

APP_ABI := armeabi-v7a

配置Android.mk

LOCAL_SRC_FILES := syscall.s

命令行下执行ndk-build即可将汇编源文件编译到目标共享库。
